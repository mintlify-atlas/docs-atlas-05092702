SpeedyBot's type definitions for core functionality.

## Configuration types

### SpeedyConfig

Configuration object for SpeedyBot initialization.

```typescript
type SpeedyConfig = Partial<{
  allowedEmails: string[];
}>
```

<ResponseField name="allowedEmails" type="string[]">
  Whitelist of email addresses allowed to interact with the bot
</ResponseField>

## Error types

### SpeedyError

Error object passed to error handlers.

```typescript
type SpeedyError = {
  e: unknown;              // Original error
  message: string;         // Error message
  status?: string | number; // HTTP status or error code
  roomId?: string;         // Room where error occurred
}
```

**Example:**

```typescript
Bot.captureError((error: SpeedyError) => {
  console.error(`[${error.status}] ${error.message}`);
  if (error.roomId) {
    Bot.sendTo(error.roomId, 'An error occurred');
  }
});
```

## Message types

### Message

Any valid message content.

```typescript
type Message = string | number | Card | SpeedyCard
```

### Card

Raw Adaptive Card JSON specification.

```typescript
type Card = {
  type: string;
  body: unknown[];
  actions?: any[];
  $schema: string;
  version: string;
}
```

### MessageTypes

Incoming message type classification.

```typescript
type MessageTypes = "card" | "file" | "text"
```

### MessageResponse

Response from sending a message.

```typescript
type MessageResponse = {
  id: string;           // Message ID
  roomId: string;       // Room ID
  roomType: string;     // "direct" | "group"
  text: string;         // Plain text
  markdown?: string;    // Markdown text
  html?: string;        // HTML rendering
  personId: string;     // Sender person ID
  personEmail: string;  // Sender email
  created: string;      // ISO timestamp
  updated?: string;     // ISO timestamp (for edited messages)
  files?: string[];     // File URLs
}
```

### ToMessage

Payload for creating a message.

```typescript
type ToMessage = {
  roomId?: string;
  parentId?: string;        // For threaded replies
  toPersonId?: string;
  toPersonEmail?: string;
  text?: string;
  markdown?: string;
  files?: string[];
  attachments?: {
    contentType?: string;
    content?: unknown;      // Rendered card
  }[];
}
```

## File types

### SpeedyFile

File metadata.

```typescript
type SpeedyFile = {
  url: string;         // File URL
  name: string;        // File name
  extension: string;   // Extension without dot
  contentType: string; // MIME type
  bytes: number;       // File size
}
```

### SpeedyFileUtils

File utilities with data accessor.

```typescript
type SpeedyFileUtils = SpeedyFile & {
  getData<T = unknown>(): Promise<T>;
}
```

**Example:**

```typescript
if ($.file?.extension === 'json') {
  const data = await $.file.getData<MyType>();
  console.log(`${$.file.name}: ${$.file.bytes} bytes`);
}
```

## Data types

### AttachedData

Generic data object for card submissions.

```typescript
type AttachedData = {
  [key: string]: boolean | number | string | Record<string, any> | AttachedData;
}
```

**Example:**

```typescript
type MyFormData = AttachedData & {
  username: string;
  priority: string;
  notify: boolean;
};

Bot.addStep<MyFormData>(async ($) => {
  if ($.data) {
    const { username, priority, notify } = $.data;
    // Type-safe access
  }
  return $.next;
});
```

## Middleware types

### Middleware

Middleware handler function.

```typescript
type Middleware<T = AttachedData> = (
  $: $Magic<T>
) => boolean | Promise<boolean>
```

**Returns:** `true` or `$.next` to continue, `false` or `$.end` to stop

### $Magic

The magic $ parameter. See [$ parameter documentation](/api/magic-dollar) for details.

```typescript
type $Magic<T = AttachedData> = IncomingItem<T> & HelperMethods
```

### IncomingItem

Union of possible incoming message types.

```typescript
type IncomingItem<T> = TextData<T> | FileData<T> | CardData<T>
```

type RequestOps = {

## Utility types

### DeepPartial

Recursive partial type.

```typescript
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
}
```

**Example:**

```typescript
type Config = {
  server: {
    host: string;
    port: number;
  };
};

const partialConfig: DeepPartial<Config> = {
  server: { host: 'localhost' } // port is optional
};
```
