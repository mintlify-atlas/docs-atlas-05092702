---
title: SpeedyBot class
description: Core SpeedyBot class for building chatbots
---

The `SpeedyBot` class is the main interface for building and managing your bot. It handles webhooks, middleware, and all interactions with the Webex API.

## Constructor

```typescript
const Bot = new SpeedyBot(token?: string, makeRequest?: CoreMakerequest)
```

<ParamField path="token" type="string" optional>
  Bot access token. Can also be set later with `setToken()`
</ParamField>

<ParamField path="makeRequest" type="CoreMakerequest" optional>
  Custom request handler for testing or proxying
</ParamField>

## Configuration methods

### setToken

Sets the bot token for API authentication.

```typescript
Bot.setToken(token: string): SpeedyBot
```

<ParamField path="token" type="string" required>
  Your bot access token from the Webex developer portal
</ParamField>

**Returns:** The SpeedyBot instance for chaining

### getToken

Retrieves the current bot token.

```typescript
Bot.getToken(): string
```

**Returns:** The configured bot token

### setFallbackText

Sets fallback text for Adaptive Cards when the client doesn't support them.

```typescript
Bot.setFallbackText(text: string): void
```

<ParamField path="text" type="string" required>
  Fallback message text
</ParamField>

## Middleware registration

### addStep

Adds a middleware handler to the processing chain.

```typescript
Bot.addStep<T = AttachedData>(middleware: Middleware<T>): void
```

<ParamField path="middleware" type="Middleware<T>" required>
  Handler function that receives the `$` parameter
</ParamField>

**Example:**

```typescript
Bot.addStep(async ($) => {
  await $.send('Hello!');
  return $.next;
});
```

### addStepSequence

Adds multiple middleware handlers at once.

```typescript
Bot.addStepSequence<T = AttachedData>(middlewares: Middleware<T>[]): void
```

<ParamField path="middlewares" type="Middleware<T>[]" required>
  Array of middleware functions
</ParamField>

### insertStepToFront

Inserts middleware at the beginning of the chain.

```typescript
Bot.insertStepToFront(middleware: Middleware): void
```

<ParamField path="middleware" type="Middleware" required>
  Handler to insert at the front
</ParamField>

## Message sending

### sendTo

Sends a message to a room, email, or person.

```typescript
Bot.sendTo(
  emailOrRoomId: string | { personId: string },
  message: Message
): Promise<MessageResponse>
```

<ParamField path="emailOrRoomId" type="string | { personId: string }" required>
  Target email, room ID, or person ID object
</ParamField>

<ParamField path="message" type="Message" required>
  String, number, Card, or SpeedyCard to send
</ParamField>

**Returns:** Promise resolving to message details

**Example:**

```typescript
await Bot.sendTo('user@example.com', '**Hello!**');
await Bot.sendTo('room-id', Bot.card().addTitle('Card'));
```

### send (internal)

Internal method for sending messages.

```typescript
Bot._send(payload: ToMessage): Promise<MessageResponse>
```

### replyTo

<ParamField path="messageId" type="string" required>

</ParamField>

<ParamField path="messageObj" type="MessageResponse" optional>
  Message object containing roomId and id
</ParamField>

<ParamField path="message" type="string" required>
  Reply text
</ParamField>

**Returns:** Promise resolving to reply message details

### editMessage

<ParamField path="opts" type="object" optional>

### peekFile

## Error handling

### captureError

Registers a global error handler.

```typescript
Bot.captureError(handler: (error: SpeedyError) => void | unknown): void
```

<ParamField path="handler" type="function" required>
  Error handling function
</ParamField>

**Example:**

```typescript
Bot.captureError((error) => {
  console.error('Bot error:', error.message);
  if (error.roomId) {
    Bot.sendTo(error.roomId, 'An error occurred');
  }
});
```


<ParamField path="arr" type="string[]" required>
  Array of "key=value" strings
</ParamField>
