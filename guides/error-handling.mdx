Proper error handling ensures your bot remains responsive and helpful even when things go wrong.

## Basic error handler

Use `Bot.captureError()` (alias for `setErrorHandler`) to catch errors from any step:

```typescript
import { SpeedyBot } from 'speedybot';

const Bot = new SpeedyBot();

// Define error handler
Bot.captureError(async (error) => {
  const { message, roomId, status } = error;
  
  if (roomId) {
    await Bot.sendTo(
      roomId,
      `Whoops, there was a problem: ${message}`
    );
  }
  
  // Log for debugging
  console.error('Bot Error:', { message, status });
});

export default Bot;
```

### Log errors for debugging

```typescript
Bot.captureError(async (error) => {
  const { e, message, status, roomId } = error;
  
  // Log to console
  console.error('Error occurred:', {
    message,
    status,
    roomId,
    stack: e instanceof Error ? e.stack : 'No stack trace',
  });
  
  // Send to monitoring service
  await logToMonitoring({
    severity: 'error',
    message,
    context: { roomId, status },
  });
  
  // Notify user
  if (roomId) {
    await Bot.sendTo(roomId, 'An error occurred. Support has been notified.');
  }
});
```

### Distinguish error types

```typescript
Bot.captureError(async (error) => {
  const { e, message, roomId, status } = error;
  
  // Network/API errors
  if (status === 429) {
    if (roomId) {
      await Bot.sendTo(
        roomId,
        'Rate limit exceeded. Please try again in a moment.'
      );
    }
    return;
  }
  
  // Authentication errors
  if (status === 401 || status === 403) {
    console.error('Authentication error - check bot token');
    return;
  }
  
  // File handling errors
  if (message.includes('file')) {
    if (roomId) {
      await Bot.sendTo(
        roomId,
        'There was an issue processing the file. Please try again.'
      );
    }
    return;
  }
  
  // Default error message
  if (roomId) {
    await Bot.sendTo(roomId, 'Something went wrong. Please try again.');
  }
});
```

## Handling errors in steps

You can also use try-catch within individual steps:

```typescript
Bot.addStep(async ($) => {
  try {
    // Risky operation
    const result = await externalAPI.call($.text);
    await $.send(`Result: ${result}`);
  } catch (error) {
    console.error('API call failed:', error);
    await $.send('Sorry, the external service is unavailable.');
  }
  return $.next;
});
```

## Complete error handling example

```typescript
import { SpeedyBot, SpeedyError } from 'speedybot';

const Bot = new SpeedyBot();

// Global error handler
Bot.captureError(async (error: SpeedyError) => {
  const { e, message, status, roomId } = error;
  
  // Log to monitoring service
  console.error('Bot Error:', {
    message,
    status,
    roomId,
    timestamp: new Date().toISOString(),
  });
  
  // Send error monitoring
  if (process.env.SENTRY_DSN) {
    // Sentry.captureException(e);
  }
  
  // Notify user if possible
  if (roomId) {
    let userMessage = 'An unexpected error occurred.';
    
    if (status === 429) {
      userMessage = 'Too many requests. Please try again in a moment.';
    } else if (status === 403 || status === 401) {
      userMessage = 'Authentication error. Please contact support.';
    } else if (message.includes('file')) {
      userMessage = 'There was an issue processing the file.';
    }
    
    const errorCard = Bot.errorCard('Error', userMessage);
    await Bot.sendTo(roomId, errorCard);
  }
});

// Steps with error handling
Bot.addStep(async ($) => {
  if ($.file) {
    try {
      const data = await $.file.getData();
      await $.send('File processed!');
    } catch (error) {
      await $.send('Could not process file. Please try a different format.');
      throw error; // Will be caught by global handler
    }
  }
  return $.next;
});

Bot.addStep(async ($) => {
  if ($.text) {
    try {
      const result = await externalAPI.process($.text);
      await $.send(`Result: ${result}`);
    } catch (error) {
      console.error('API Error:', error);
      await $.send('The external service is temporarily unavailable.');
      // Don't throw - handled locally
    }
  }
  return $.next;
});

export default Bot;
```

## Error handling checklist

- [ ] Global error handler configured with `Bot.captureError()`
- [ ] User-friendly error messages (no technical jargon)
- [ ] Errors logged for debugging
- [ ] Monitoring/alerting integrated
- [ ] Different error types handled appropriately
- [ ] Rate limiting errors handled gracefully
- [ ] File operation errors caught
- [ ] External API failures have fallbacks
- [ ] Webhook validation errors handled
- [ ] Error cards used for better UX

<Info>
The error handler is your safety net. Always implement it to prevent your bot from silently failing.
</Info>

<Warning>
Avoid exposing sensitive error details (API keys, stack traces, internal paths) to end users. Log these securely instead.
</Warning>
