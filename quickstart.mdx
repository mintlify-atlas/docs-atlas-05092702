# Quickstart

Get a working SpeedyBot up and running in minutes. This guide will walk you through creating a simple bot that responds to messages and handles interactive cards.

## Prerequisites

Before you begin, you'll need:

- Node.js 18.11.0 or higher
- A WebEx account
- A bot access token from [developer.webex.com](https://developer.webex.com)

## Create your first bot

<Steps>
  <Step title="Install SpeedyBot">
    Create a new project and install SpeedyBot:

    <CodeGroup>
    ```bash npm
    npm install speedybot
    ```

    ```bash yarn
    yarn add speedybot
    ```

    ```bash pnpm
    pnpm add speedybot
    ```

    ```bash bun
    bun add speedybot
    ```
    </CodeGroup>
  </Step>

  <Step title="Create your bot file">
    Create a new file called `bot.ts` with the following code:

    ```ts bot.ts
    import { SpeedyBot } from "speedybot";

    const Bot = new SpeedyBot();

    // Handle file uploads
    Bot.addStep(async ($) => {
      if ($.file) {
        const { name, extension, contentType } = $.file;
        await $.send(
          `You uploaded "${name}", a *.${extension} file [${contentType}]`
        );
        // Fetch raw bytes if needed
        // const fileData = await $.file.getData();
      }
      return $.next;
    });

    // Respond to ping/pong
    Bot.addStep(async ($) => {
      if ($.text) {
        const lowered = $.text.toLowerCase();
        if (lowered === "ping") {
          await $.send("pong");
          return $.end;
        } else if (lowered === "pong") {
          await $.send("ping");
          return $.end;
        }
      }
      return $.next;
    });

    // Show an interactive card
    Bot.addStep(async ($) => {
      if ($.text?.toLowerCase() === "show-card") {
        const card = $.card()
          .addTitle("SpeedyBot Card")
          .addText("This is an interactive card")
          .addChips(["Option 1", "Option 2", "Option 3"])
          .addTable([
            ["Feature", "Status"],
            ["Zero dependencies", "✅"],
            ["TypeScript", "✅"],
          ]);
        await $.send(card);
        return $.end;
      }
      return $.next;
    });

    // Main greeting
    Bot.addStep(async ($) => {
      if ($.text && !$.file) {
        const utterances = [
          `Heya how's it going $[name]?`,
          `Hi there, $[name]!`,
          `Hiya $[name]!`,
        ];
        const template = { name: $.author.name };
        await $.send($.fillTemplate(utterances, template));

        const card = $.card()
          .addTitle("🤖 SpeedyBot")
          .addText("Try these commands:")
          .addChips([
            { title: "ping 🏓", value: "ping" },
            { title: "🏓 pong", value: "pong" },
            { title: "Show Card 🎴", value: "show-card" },
          ]);
        await $.send(card);
      }
      return $.next;
    });

    // Handle card submissions
    Bot.addStep(async ($) => {
      if ($.data) {
        const dataSnippet = $.buildDataSnippet($.data);
        await $.send("This data was submitted:");
        await $.send(dataSnippet);
      }
      return $.next;
    });

    // Error handling
    Bot.captureError(async (payload) => {
      const { roomId } = payload;
      if (roomId) {
        await Bot.sendTo(
          roomId,
          `Whoops, there was a problem: ${payload.message}`
        );
      }
    });

    export default Bot;
    ```
  </Step>

  <Step title="Set your bot token">
    Set your bot token as an environment variable:

    ```bash
    export BOT_TOKEN="your_bot_token_here"
    ```

    <Tip>
      Get your bot token from [developer.webex.com](https://developer.webex.com) by creating a new bot.
    </Tip>
  </Step>

  <Step title="Run your bot">
    For local development, you can use the SpeedyBot CLI to quickly set up and run your bot:

    ```bash
    npx speedybot setup
    ```

    The CLI will guide you through:
    - Setting up webhooks
    - Configuring your bot token
    - Running a local development server
  </Step>
</Steps>

## Understanding the code

### Step execution

SpeedyBot processes incoming events through a series of steps. Each step:

- Receives a `$` parameter with utilities and event data
- Can be synchronous or asynchronous
- Must return `$.next` to continue or `$.end` to stop

```ts
Bot.addStep(async ($) => {
  // Your logic here
  return $.next; // Continue to next step
});
```

### The `$` parameter

The `$` object provides everything you need:

- `$.text` - The user's message text
- `$.author` - User information (name, email, id)
- `$.file` - File upload data (if present)
- `$.data` - Form submission data (if present)
- `$.send()` - Send messages
- `$.reply()` - Reply to messages
- `$.card()` - Create cards

### SpeedyCards

Create rich, interactive cards with a fluent API:

```ts
const card = $.card()
  .addTitle("My Card")
  .addText("Some description")
  .addChips(["Action 1", "Action 2"])
  .addTextInput("Enter your name")
  .addButton("Submit", "submit_action");

await $.send(card);
```

## Common patterns

### Exact keyword matching

```ts
Bot.exact("help", async ($) => {
  await $.send("Here's how I can help...");
  return $.end;
});
```

### Fuzzy matching

```ts
Bot.contains(["hello", "hi", "hey"], async ($) => {
  await $.send(`Hello ${$.author.name}!`);
  return $.end;
});
```

### Regex matching

```ts
Bot.regex(/^order #(\d+)$/i, async ($) => {
  // Handle order lookup
  return $.next;
});
```

### Message threads

```ts
Bot.addStep(async ($) => {
  await $.thread([
    "Main message",
    "Reply 1",
    "Reply 2",
    "Reply 3",
  ]);
  return $.next;
});
```
